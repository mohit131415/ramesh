üìã Overview

The User Profile API allows users to manage their personal profile information including personal details, profile pictures, and track profile completion status.

Base URL

https://your-domain.com/api/public/user/profile

Key Features

  • Create and update user profiles
  • Upload and manage profile pictures
  • Track profile completion percentage
  • Email uniqueness validation
  • Phone number integration from user account

üîê Authentication

Important: All endpoints require authentication via JWT token in the Authorization header.

Header Format

Authorization: Bearer YOUR_JWT_TOKEN

User Identification

The user is automatically identified from the JWT token. No need to pass user_id in requests.

üöÄ API Endpoints

POST /api/public/user/profile

Description: Create a new user profile

Request Parameters

Parameter Type Required Description
first_name string Optional User's first name (max 100 characters)
last_name string Optional User's last name (max 100 characters)
gender enum Optional male, female, other, prefer_not_to_say
email string Optional Valid email address (must be unique)
date_of_birth date Optional Format: YYYY-MM-DD
profile_picture file Optional Image file (JPEG, PNG, GIF, WebP, max 5MB)

Success Response (201)

{ "status": "success", "message": "Profile created successfully", "data": { "id": 1, "user_id": 123, "first_name": "John", "last_name": "Doe", "gender": "male", "email": "john.doe@example.com", "date_of_birth": "1990-01-15", "profile_picture": "uploads/profile_pictures/profile_123456.jpg", "profile_picture_url": "https://your-domain.com/uploads/profile_pictures/profile_123456.jpg", "phone_number": "+1234567890", "user_status": "active", "last_login_at": "2024-01-15 10:30:00", "created_at": "2024-01-15 10:30:00", "updated_at": "2024-01-15 10:30:00" } }

GET /api/public/user/profile

Description: Retrieve the authenticated user's profile

Success Response (200)

PUT /api/public/user/profile

Description: Update the authenticated user's profile

Request Parameters

Same as create profile. Only send fields you want to update.

Success Response (200)

{ "status": "success", "message": "Profile updated successfully", "data": { // Updated profile data (same structure as get profile) } }

GET /api/public/user/profile/completion

Description: Get profile completion status and percentage

Success Response (200)

{ "status": "success", "message": "Profile completion status retrieved successfully", "data": { "completion_percentage": 83, "missing_fields": ["profile_picture"], "completed_fields": ["first_name", "last_name", "gender", "email", "date_of_birth"] } }

import { useState, useEffect } from 'react'; const useUserProfile = () => { const [profile, setProfile] = useState(null); const [loading, setLoading] = useState(false); const [error, setError] = useState(null); const fetchProfile = async () => { setLoading(true); try { const response = await fetch('/api/public/user/profile', { headers: { 'Authorization': `Bearer ${localStorage.getItem('token')}` } }); const data = await response.json(); if (data.status === 'success') { setProfile(data.data); } else { setError(data.message); } } catch (err) { setError('Failed to fetch profile'); } finally { setLoading(false); } }; const updateProfile = async (updates) => { setLoading(true); try { const formData = new FormData(); Object.keys(updates).forEach(key => { if (updates[key] !== null && updates[key] !== undefined) { formData.append(key, updates[key]); } }); const response = await fetch('/api/public/user/profile', { method: 'PUT', headers: { 'Authorization': `Bearer ${localStorage.getItem('token')}` }, body: formData }); const data = await response.json(); if (data.status === 'success') { setProfile(data.data); return data; } else { setError(data.message); throw new Error(data.message); } } catch (err) { setError(err.message); throw err; } finally { setLoading(false); } }; useEffect(() => { fetchProfile(); }, []); return { profile, loading, error, updateProfile, refetch: fetchProfile }; };

‚ö†Ô∏è Error Handling

HTTP Status Codes

  • 200: Success
  • 201: Created successfully
  • 400: Bad request / Validation error
  • 401: Unauthorized (invalid/missing token)
  • 404: Profile not found
  • 422: Validation failed
  • 500: Internal server error

Error Response Format

{ "status": "error", "message": "Validation failed", "errors": { "email": "Email is already in use", "date_of_birth": "Date of birth must be in format YYYY-MM-DD" }, "data": null }

Common Validation Errors

Field Error Solution
email Email is already in use Use a different email address
email Invalid email format Provide a valid email address
date_of_birth Date of birth must be in format YYYY-MM-DD Use correct date format
date_of_birth Date of birth cannot be in the future Use a past date
gender Gender must be one of: male, female, other, prefer_not_to_say Use valid gender value
profile_picture Invalid file type Use JPEG, PNG, GIF, or WebP
profile_picture File size too large Use file smaller than 5MB

üìù Notes for Frontend Developers

  • File Uploads: Use FormData for requests with file uploads
  • Profile Pictures: Always check for both 'profile_picture' and 'profile_picture_url' fields
  • Phone Number: Phone number comes from the user account, not the profile
  • Email Uniqueness: Handle email uniqueness errors gracefully
  • Profile Completion: Use the completion endpoint to show progress to users
  • Optional Fields: All profile fields are optional - handle null/empty values
  • Date Format: Always use YYYY-MM-DD format for dates
  • Error Handling: Always check the 'status' field in responses
